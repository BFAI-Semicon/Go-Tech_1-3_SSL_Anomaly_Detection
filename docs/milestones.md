# 2026年度 月次マイルストーン（2026年7月〜2027年2月）

---

## 前半：PatchCore 蒸留 ＋ 既存手法ベース整備（2026/07〜10）

### 2026年7月 — 基盤整備・PatchCore 教師ベースライン

- **目的**：再現可能な実験基盤と蒸留の教師となる PatchCore ベースライン
  ＋速度/メモリ計測を確立し、生徒モデルの設計に着手
- **主タスク**：
  - 単一分割（hold-out）と評価スクリプト整備
    （AUROC/AUPRC/F1 ＋ FPS・GPUメモリ計測を組み込み）
  - PatchCore 教師の構築（coreset 率スイープでメモリバンク／精度の関係を取得）
  - 簡易な信頼区間（Confidence Interval, CI；点推定 ＋ 小規模 bootstrap）で
    教師ベースラインを先行確定
  - 生徒モデル設計（軽量 CNN / 小型バックボーン、メモリバンク不要の forward 推論）
- **主な狙い**：実験パイプライン雛形、PatchCore 教師ベースライン
  ＋速度/メモリ表（簡易CI付き）、生徒モデル設計案

### 2026年8月 — PatchCore 蒸留プロトタイプ

- **目的**：評価基盤を本格化しつつ、教師 PatchCore から軽量な生徒モデルへの
  蒸留を動作させる
- **主タスク**：
  - k-fold（正常分割）と bootstrap CI を本格化（全異常をテスト使用、異常116枚対策）
  - 特徴蒸留（patch 埋め込み回帰）／異常マップ蒸留（異常スコアマップ再現）の実装
  - 生徒の FPS・GPU メモリ・精度のトレードオフを初期計測
- **主な狙い**：k-fold＋CI付き評価基盤、蒸留プロトタイプ、
  教師 vs 生徒の精度/速度/メモリ比較表

### 2026年9月 — 蒸留深化 ＋ 既存手法・前後処理補完

- **目的**：蒸留モデルの精度を詰めつつ、比較対象の既存手法・前後処理を押さえる
- **主タスク**：
  - 欠陥サイズ別（小/中/大）の精度劣化を評価（中間報告 評価②に対応）
  - 蒸留方式・生徒構造・coreset 率のアブレーション
  - 既存手法横並び（CFlow/DFM/DFKDE/EfficientAD/DRAEM/AnomalyDINO/WinCLIP）と
    前後処理スイープ（補完・簡易）
- **主な狙い**：蒸留アブレーション表、欠陥サイズ別性能、既存手法ベンチ

### 2026年10月 — 蒸留モデル確定・前半総括

- **目的**：軽量・高速な蒸留モデルを確定し、MIIC 上の上限性能と固定構成をまとめる
- **主タスク**：
  - 蒸留モデルの最終調整と精度/速度/メモリのトレードオフ表確定
  - score-level fusion（平均/重み付き/rank fusion/学習型ゲート）で上限性能を補完
  - 採用特徴抽出器・前後処理・融合方式の固定、
    残存する誤検出・見逃しの類型化（→後半HITL入力）
- **主な狙い**：前半最終レポート（蒸留モデル、上限性能、固定構成、
  誤検出/見逃し類型）

## 後半：自己教師あり学習による Promptable Patch Retrieval（2026/11〜2027/02）

### 2026年11月 — SSL特徴抽出器＋ストア基盤

- **目的**：固定ViTの特徴抽出と特徴量ストアを立ち上げ
- **主タスク**：
  - DINOv2埋め込み＋MAE再構成誤差のパッチ単位生成
  - FAISS等のkNN基盤（工程/材料/装置タグでメタ分割）構築
  - 独自データのタイル化・パッチ化（前半の前処理を共通化）
- **主な狙い**：特徴抽出＋ストアのプロトタイプ、初期インデックス設計

### 2026年12月 — 一次検出＋特徴抽出器比較

- **目的**：一次検出の動作確立と特徴抽出器の優劣を定量化
- **主タスク**：
  - 埋め込み逸脱（PatchCore on DINOv2）＋MAE再構成誤差の融合
  - 特徴抽出器比較（ImageNet教師ありCNN vs DINO/DINOv2 vs DINO+MAE vs C-RADIOv2）
  - 工程内汎化 vs 工程横断（Si↔化合物）の劣化曲線を取得
- **主な狙い**：一次検出ベンチ、特徴抽出器比較レポート、汎化劣化曲線

### 2027年1月 — HITL（ROI＋言語）＋補正レイヤ

- **目的**：人間フィードバックを推論時条件として統合
- **主タスク**：
  - ROI注釈＋自然言語コメントのUI/運用プロトコル
  - LLMによるJSON構造化（scope/judgment/priority）＋スキーマ検証・監査ログ
  - 補正レイヤ（スコア再重み付け/閾値適応/ラベル上書き）の実装
- **主な狙い**：HITLループのE2E動作、JSON構造化スキーマ、補正レイヤ実装

### 2027年2月 — 評価総括＋計算資源検証

- **目的**：後半の検証軸を出し切り、年度成果を統合
- **主タスク**：
  - 評価軸の網羅：ROIのみ/言語のみ/併用、スコア補正 vs ラベル上書き、改善量
  - DGX Spark実測（パッチ数×スケール×ストア規模スイープ）とPC GPU比較
  - DGX到着遅延に備えたPC GPU縮約設定で代替実行
- **主な狙い**：後半最終レポート＋年度総括、計算資源要件の定量根拠
